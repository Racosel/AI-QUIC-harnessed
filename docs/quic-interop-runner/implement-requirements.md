### 容器退出状态码规范 (Container Exit Status Codes)
> **全局约束：** 客户端/服务器容器在运行结束时，**必须 (MUST)** 严格按照以下标准返回对应的退出状态码 (Exit Status Code)，以便 Interop Test Runner 框架准确判定测试结果。

* **正常成功完成 (Success)**
  * **状态码：** `0`
  * **触发条件：** 传输任务按照预期完全结束，且无任何内部错误。

* **执行发生错误 (Error)**
  * **状态码：** `1`
  * **触发条件：** 在测试执行过程中出现运行异常、文件传输中止、或协议层面验证失败等错误情况。

* **不支持该测试用例 (Unsupported Testcase)**
  * **状态码：** `127`
  * **触发条件 (强制/MUST)：** 当解析到 `TESTCASE` 环境变量的值为当前实现不支持的测试用例时。
  * **机制意义：** 直接退出并返回 `127`，测试框架会将其识别为“未支持”，而非“测试失败”。这使得框架可以随时添加新的测试用例，而不会破坏或误判现有的稳定实现。
