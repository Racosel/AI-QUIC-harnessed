
## 测试用例定义 (Test Cases)
> **全局说明：** 除非另有说明，测试用例均使用 **HTTP/0.9** 进行文件传输。

* **版本协商 (Version Negotiation)**
  * **参数赋值：** `TESTCASE="versionnegotiation"`
  * **描述：** 测试服务器在响应未知的 QUIC 版本号时是否发送版本协商数据包。客户端应使用不支持的版本号发起连接，并在收到版本协商数据包时中止连接。*(注：由于 #20，当前处于禁用状态)*

* **握手 (Handshake)**
  * **参数赋值：** `TESTCASE="handshake"`
  * **描述：** 测试握手是否成功完成。客户端需建立单个 QUIC 连接到服务器，并下载一个或多个小文件。服务器在此阶段不应发送 Retry 数据包。

* **传输 (Transfer)**
  * **参数赋值：** `TESTCASE="transfer"`
  * **描述：** 测试流量控制和流多路复用。客户端应设置较小的初始流级和连接级流量控制窗口，在传输约 1 MB 文件期间触发窗口增加。客户端需建立单连接，多流并发下载。

* **ChaCha20**
  * **参数赋值：** `TESTCASE="chacha20"`
  * **描述：** 客户端和服务器预期**仅**提供 ChaCha20 作为加密套件。随后客户端下载文件。

* **密钥更新 (KeyUpdate)**
  * **参数赋值：** `TESTCASE="keyupdate"` *(仅限客户端)*
  * **描述：** 客户端需确保在连接早期（传输前 1 MB 期间）发生密钥更新。由哪个对等端发起更新无关紧要。

* **重试 (Retry)**
  * **参数赋值：** `TESTCASE="retry"`
  * **描述：** 测试服务器是否能够生成 Retry 数据包，以及客户端是否能在 Initial 数据包中使用 Retry 数据包提供的 Token。

* **会话恢复 (Resumption)**
  * **参数赋值：** `TESTCASE="resumption"`
  * **描述：** 测试无 0-RTT 的会话恢复。客户端建立连接下载首个文件，服务器提供会话票证。客户端关闭连接后，使用票证恢复连接并下载剩余文件。

* **0-RTT**
  * **参数赋值：** `TESTCASE="zerortt"`
  * **描述：** 测试 0-RTT。建立连接下载首个文件后，服务器提供允许 0-RTT 的票证。客户端关闭连接，建立 0-RTT 连接并请求剩余文件。

* **HTTP/3**
  * **参数赋值：** `TESTCASE="http3"`
  * **描述：** 测试简单的 HTTP/3 连接。客户端使用 HTTP/3 并行请求和传输多个文件。

* **握手丢包 (Handshake Loss)**
  * **参数赋值：** `TESTCASE="multiconnect"`
  * **描述：** 测试握手在高丢包环境下的弹性。客户端建立多个连接（顺序或并行），并使用每个连接下载单个文件。

* **QUIC v2 (V2)**
  * **参数赋值：** `TESTCASE="v2"`
  * **描述：** 客户端在 QUIC v1 中开始连接，其传输参数包含 QUIC v2 (`0x6b3343cf`)。服务器通过兼容协商选择 v2。客户端在 QUIC v2 中下载小文件。

* **端口重绑定 (Port Rebinding)**
  * **参数赋值：** `TESTCASE="rebind-port"`
  * **描述：** 模拟 NAT 在握手后更改客户端的源端口。服务器应执行路径验证。

* **地址重绑定 (Address Rebinding)**
  * **参数赋值：** `TESTCASE="rebind-addr"`
  * **描述：** 模拟 NAT 在握手后更改客户端的 IP 地址。服务器应执行路径验证。

* **连接迁移 (Connection Migration)**
  * **参数赋值：** `TESTCASE="connectionmigration"`
  * **描述：** 服务器在握手期间向客户端提供其首选地址。客户端主动迁移到其中一个地址。
